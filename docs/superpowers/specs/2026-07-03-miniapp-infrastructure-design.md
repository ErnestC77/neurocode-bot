# Mini App — инфраструктура (подпроект 1 из 3) — design spec

Дата: 2026-07-03
Статус: approved

## Контекст

Пользователь хочет перенести все взаимодействия бота «Диагностика нейрокода» в Telegram Mini App: квиз должен «выглядеть дорого» (палитра золото `#e8c96a` на кнопках, тёмно-синий фон `#162a48`, белый текст), плюс админ-панель (сейчас `/settings` + `/export_leads` в чате) должна переехать туда же, включая просмотр лидов.

Это большой проект, разбит на 3 независимых подпроекта (каждый — своя спека/план):
1. **Инфраструктура Mini App** (эта спека) — обязательный фундамент.
2. Экраны воронки (квиз, результат, оплата) — отдельно, позже.
3. Админ-экран (настройки + лиды) — отдельно, позже, независимо от (2).

## Референс

`C:\Users\mccaq\IdeaProjects\barbershop-bot` (ветка `feature/telegram-mini-app`) — у пользователя уже есть рабочий Mini App с почти идентичной золотой палитрой (`#e8c873` на тёмном) и проверенной архитектурой. Эта спека сознательно копирует его паттерны, а не изобретает новые:

- Бэкенд: FastAPI, один процесс/сервис, `uvicorn asgi:app`.
- Бот (aiogram long-polling) — фоновая задача внутри FastAPI `lifespan` (паттерн `bot_lifecycle` из `app/api/app.py`), не отдельный процесс.
- Auth: `app/api/auth.py::parse_and_validate_init_data` — HMAC-SHA256 проверка `initData` от Telegram WebApp SDK. Переносится почти без изменений.
- SPA монтируется в FastAPI ПОСЛЕ `/api/*` роутов (Starlette матчит по порядку регистрации — `/api` не может быть перекрыт статикой). `index.html` — `Cache-Control: no-cache` (иначе Telegram WebView держит старую сборку до ручной очистки кэша).
- Admin-роуты в референсе: FastAPI-роутер + `Depends(current_admin)` + Pydantic-схемы In/Out + сервисный слой — паттерн для будущего подпроекта 3, здесь просто фиксируется как договорённость.

## Технологический стек

- **Бэкенд:** FastAPI (замена aiohttp). `pydantic` для валидации схем запросов/ответов будущих подпроектов 2/3 (в этой спеке — только скелет).
- **Frontend:** React + TypeScript + Vite + `react-router-dom` (hash-роутинг — deep links в URL fragment, невидимы серверу, любой путь резолвится в один и тот же `index.html`).
- **UI-стек:** Tailwind CSS (вся стилизация, включая палитру) + `@telegram-apps/telegram-ui` (нативные для Mini App компоненты — кнопки, списки, модалки) + `shadcn/ui` точечно, для того, чего нет в telegram-ui (кастомные карточки квиза, прогресс-бар в подпроекте 2). **Без MUI** — конфликтует по дизайн-языку и стилевому движку с Tailwind/shadcn, дублирует функциональность.
- Тесты фронтенда: `vitest` (как в референсе), если понадобятся в этой задаче — для чистых утилит (например, парсинг initData на фронте, если такой появится).

## Auth

- `services/telegram_auth.py` (новый модуль) — прямой порт `parse_and_validate_init_data` из barbershop-bot: HMAC-SHA256 над отсортированными `key=value` парами initData, ключ — `HMAC_SHA256(key="WebAppData", msg=bot_token)`; отклоняет протухшие payload'ы по `auth_date` (`max_age_seconds`, дефолт как в референсе — 86400).
- FastAPI-зависимости: `current_client` (валидный initData → tg_id, не более), `current_admin` (дополнительно проверяет `await services.settings.is_authorized_admin(tg_id, config)` — **переиспользуем существующую функцию**, не дублируем логику запасного env-админа).
- Секрет — тот же `BOT_TOKEN`, что уже есть в конфиге.

## Раздача SPA + запуск

- `frontend/dist` (Vite build) монтируется в FastAPI как `StaticFiles(html=True)`, копия `_SpaStaticFiles` из референса (no-cache только на HTML-точку входа, хэшированные ассеты кэшируются нормально).
- Запуск Mini App — нативная **Menu Button** Telegram (`bot.set_chat_menu_button(MenuButtonWebApp(text=..., web_app=WebAppInfo(url=...)))`), вызывается один раз при старте процесса (в lifespan). Это постоянная кнопка у поля ввода — стандартный способ запуска Mini App, не разовая inline-кнопка в сообщении.
- Внутри этой задачи — один тестовый экран (`/`), подтверждающий: initData провалидирован, `tg_id` получен и отображён, роут `/api/ping`-эквивалент отвечает. Реальные экраны — подпроекты 2/3.

## Что переезжает / что остаётся

- **Переезжает:** способ раздачи HTTP (aiohttp → FastAPI), способ запуска бота (из `bot.py::main()` в FastAPI lifespan), обработчик webhook ЮKassa (тот же путь `/yookassa/webhook`, тот же код `payments/webhook.py`, просто регистрируется как FastAPI-роут вместо aiohttp-роута), `/health`.
- **НЕ переезжает в этой задаче:** весь текущий чат-флоу на aiogram inline-кнопках (7 вопросов, результат, продажи, `/settings`, `/export_leads`) — продолжает работать как есть, без изменений. Экраны Mini App для этого — отдельные подпроекты.

## Деплой

Тот же Render-сервис (`srv-d93l7lm7r5hc73db981g`), тот же URL (`neurocode-bot.onrender.com`), та же Postgres. Меняются:
- `buildCommand`: `pip install -r requirements.txt && cd frontend && npm ci && npm run build`
- `startCommand`: `uvicorn asgi:app --host 0.0.0.0 --port $PORT`

Секреты и остальные env-переменные не трогаем. `render.yaml` обновляется соответственно.

## Явно вне скоупа

- Экраны воронки (квиз, результат, продажи) — подпроект 2.
- Админ-экран (настройки, лиды) — подпроект 3.
- Отключение/удаление текущего чат-флоу — произойдёт постепенно в подпроектах 2/3, не здесь.
- Апгрейд Render с free на платный план (отдельный открытый вопрос из-за бага с засыпанием — не относится к Mini App).

## Верификация

- Юнит-тест на `services/telegram_auth.py::parse_and_validate_init_data`: валидная подпись проходит, испорченная/просроченная — `InvalidInitDataError`. Чистая функция, без сети/БД — тестируется как `services/settings.py`.
- Ручная проверка: открыть Mini App через Menu Button в Telegram → тестовый экран показывает свой `tg_id` → `/health` и вебхук ЮKassa продолжают отвечать после миграции на FastAPI (регресс-проверка существующего функционала оплаты).
