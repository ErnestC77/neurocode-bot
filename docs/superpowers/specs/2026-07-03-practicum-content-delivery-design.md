# Доставка PDF-тетради и видео практикума — design spec

Дата: 2026-07-03
Статус: approved

## Контекст

Сейчас `_deliver_practicum` (`payments/delivery.py`) после оплаты практикума выдаёт только одноразовую ссылку-инвайт в закрытый канал. Нужно добавить PDF-рабочую тетрадь («Нейрокод денег Рабочая тетрадь.pdf», 123 КБ) и видео (запись, реальный размер 208 МБ — превышает лимит облачного Telegram Bot API на отправку файлов, 50 МБ).

Явно вынесено за скобки этой задачи: миграция хостинга на Selectel и подъём self-hosted `telegram-bot-api` сервера (снимает лимит до 2000 МБ) — отдельный крупный инфраструктурный проект, не начинается в рамках этой сессии. Текущее решение проектируется так, чтобы переключение на файловую доставку видео после миграции не требовало правок кода — только правки значений в `/settings`.

## Настройки (расширение реестра `services/settings.py`)

Три новых ключа в `SETTINGS`, тем же паттерном, что уже есть для `book_file_id`:

| key | Метка | Тип | Дефолт |
|---|---|---|---|
| `practicum_workbook_file_id` | 📓 File ID рабочей тетради | str | `""` |
| `practicum_video_file_id` | 🎬 File ID видео | str | `""` |
| `practicum_video_url` | 🔗 Ссылка на видео (пока файл не загружен) | str | `""` |

Дефолты пустые — как и остальные str-настройки, при пустом значении соответствующая часть доставки просто не отправляется (с логом ошибки), без падения.

## Логика доставки (`payments/delivery.py::_deliver_practicum`)

Порядок после текста M6.3 с инвайтом в канал (без изменений):

1. **PDF**: если `practicum_workbook_file_id` задан → `bot.send_document(...)`. Иначе — `logger.error`, аналогично текущему поведению `book_file_id`.
2. **Видео**: если `practicum_video_file_id` задан → `bot.send_video(...)` (нативный плеер, не `send_document` — это видео, не произвольный файл). Иначе, если `practicum_video_url` задан → сообщение с инлайн-кнопкой-ссылкой «Смотреть видео» (`url=...`). Иначе — `logger.error`, ничего не отправляется.

Приоритет `video_file_id` над `video_url` — это и есть механизм переключения: как только админ впишет `practicum_video_file_id` через `/settings` (после того как self-hosted сервер снимет лимит и видео будет реально загружено в бота), код автоматически перестаёт использовать ссылку, без единой правки кода.

## Явно вне скоупа

- Миграция на Selectel и self-hosted `telegram-bot-api` — отдельный проект.
- Сжатие видео до <50 МБ — не делаем (мы выбрали путь «ссылка → потом файл», не «сжать сейчас»).
- Валидация, что `practicum_video_url` — рабочая ссылка, или что `practicum_workbook_file_id`/`practicum_video_file_id` — валидные Telegram file_id (формат непроверяем, как и остальные str-настройки в реестре).

## Верификация

- Существующий паттерн (`book_file_id` delivery) уже покрыт только ручной проверкой — этот функционал того же типа (aiogram/DB-touching), автотестов не требует.
- Ручная проверка: тестовая оплата практикума → бот присылает текст с инвайтом, затем PDF-документ, затем сообщение со ссылкой на видео (пока `practicum_video_file_id` не задан).
